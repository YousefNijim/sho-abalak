## 1. Executive Summary

**شو عبالك؟** is a multi-tenant **e-commerce / food-delivery platform for the West Bank, Palestine**, built as an Nx + pnpm monorepo. It comprises one **NestJS + Prisma + PostgreSQL** backend, three **React Native / Expo** mobile apps (Customer, Business, Driver), and one **Next.js 14** admin dashboard. It uses a manual **area/region model instead of GPS** as a deliberate localization choice.

**Overall maturity:** The **backend is genuinely strong and end-to-end functional** for the core order lifecycle, with real auth, role guards, atomic order/payment creation, a working Socket.io gateway, and an admin control surface. The **mobile apps and admin are substantively wired to the live API** (no mock-data imports remain in the customer app's main flows). However, several production-critical pieces are **stubs that only log** (FCM push, SMS/OTP, Sentry), image uploads are **local-disk only** (no S3/Cloudinary), online payments are a **mock provider**, there is **no real customer "Stores" section** (placeholder), and a handful of **customer "profile" screens are fake/static** (change-password shows a success alert without calling the API; notifications is hardcoded UI). There is **no automated test suite running in CI beyond one E2E spec**, and ~10 real TypeScript errors persist in the customer app.

**Honest completion toward a fully launchable platform: ~62%.**
*Justification:* The hard part — a correct, authorized, real-time order domain wired across four clients — is done and verified against a live DB. The remaining ~38% is the "last mile" that gates a real launch: real notifications, real payments, durable image storage, the Stores vertical, security hardening, app-store builds, and a real deploy — none of which are trivial.

## 2. Architecture & Tech Stack

### Real stack per layer (verified in code)

| Layer | Technology (actually present) | Notes |
|---|---|---|
| **Backend framework** | NestJS (modular) | 17 feature modules in `apps/api/src` |
| **ORM / DB** | Prisma + PostgreSQL 16 | Schema complete; 12 migrations applied |
| **Auth** | JWT (`@nestjs/jwt`) + **bcryptjs** (not native bcrypt) | `JwtStrategy`, `JwtAuthGuard`, `RolesGuard`, `@Roles()`, `@CurrentUser()` |
| **Real-time** | Socket.io via `@nestjs/websockets` | `SocketGateway` with JWT handshake auth + room routing — **wired into services** |
| **API docs** | Swagger / OpenAPI at `/docs` | `DocumentBuilder` in `main.ts` |
| **Validation** | `class-validator` + global `ValidationPipe({whitelist,transform})` | Present on built modules |
| **Mobile** | React Native 0.74 / Expo SDK 51, expo-router, Zustand, TanStack Query, Axios, socket.io-client | RTL forced in `_layout.tsx`; Cairo/Montserrat fonts |
| **Admin** | Next.js 14 (App Router), Tailwind, TanStack Table, Recharts, React Query | 10 dashboard routes + login; `dir="rtl"` |
| **Shared packages** | `@shu/shared-types`, `@shu/utils`, `@shu/ui-components`, `@shu/api-client` | tokens, enums, models, order-status state machine, currency |
| **Infra (config only)** | Docker Compose (PG16 + Redis7), Dockerfiles, Nginx reverse proxy, GitHub Actions CI | Config exists; **not deployed** |

### Key architectural decisions & gotchas (verified against config + Key facts)

- **Monorepo:** Nx pinned to `19.8.4` (do not bump — registry mismatch on `@nrwl/tao`). pnpm 10 with a build allow-list in `pnpm-workspace.yaml` (`onlyBuiltDependencies`).
- **Region model instead of GPS:** Orders carry an `areaId` + snapshot `deliveryAreaName`/`deliveryAddressDetail`. `Business.lat`/`lng` columns exist but are **unused** (address-only by product decision). No maps library is integrated.
- **Design tokens are code:** `packages/ui-components/src/tokens.ts` is the single source of truth; web Tailwind mirrors them, RN reads them directly. RN primitives live behind the `@shu/ui-components/native` subpath so the web admin never pulls in react-native.
- **Compiled shared packages:** `@shu/shared-types` and `@shu/utils` must be built (`dist/index.js`) before the API runs at runtime (Node can't `require()` their `.ts`).
- **Local Postgres on host port 5433** (native PG owns 5432); API on **3001** locally, admin on **3000** (collide if both 3000).
- **bcryptjs, not bcrypt** (native binding fails on this Windows box).
- **Non-interactive Prisma migrations** via `migrate diff → migrate deploy → generate` (the shell can't answer interactive prompts).

## 6. Real-time / Sockets

Gateway: `apps/api/src/gateway/socket.gateway.ts`. **JWT verified on handshake** (`auth.token` or `query.token`); rejects on missing/invalid. Joins `user:${id}` room; BUSINESS also joins `business:owner:${id}`. **All emitters are actually called** from `OrdersService`/`DriversService` (grep-confirmed: 14 call-sites in orders, 2 in drivers).

| Event | Direction | Emitter | Implemented | Verified |
|---|---|---|---|---|
| `order:new` | Server → Business | `emitOrderNew` (on order create) | ✅ | ✅ (business dashboard listens) |
| `order:status_update` | Server → Customer & Business | `emitOrderStatusUpdate` / `...ToBusiness` | ✅ | ✅ (customer tracking listens) |
| `driver:request` | Server → Driver | `emitDriverRequest` (dispatch + PICKED_UP) | ✅ | ✅ (driver request-alert listens) |
| `driver:status_change` | Server → all (broadcast) | `emitDriverStatusChange` | ✅ | 🟡 (broadcast emitted; dashboard consumption not verified) |
| `order:driver_rejected` | Server → Business | `emitOrderDriverRejected` | ✅ | 🟡 |
| `notification:push` | Server → User (spec) | **Not emitted anywhere** | ❌ | ❌ (spec event, no code) |

> Frontend `useSocket` singleton hook exists in the apps and syncs the Zustand auth token. Real-device verification has **not** been done (no emulator in the build environment) — so socket behavior is "built + code-correct," not "observed on a phone."
